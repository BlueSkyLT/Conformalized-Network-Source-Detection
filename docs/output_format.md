# Output Format Reference

This document describes all output files generated by the conformal prediction pipeline.

## Output Directory Structure

```
results/<graph>/<test_exp_name>/pow_expected<pow_value>/
├── coverage_table.csv          # Coverage statistics
├── set_size_table.csv          # Set size statistics
├── time_cost_table.csv         # Runtime statistics
├── coverage_boxplot.pdf        # Coverage visualization
├── set_size_boxplot.pdf        # Set size visualization
├── visualizations/             # Per-sample visualizations (from visualization.py)
│   └── sample<i>_*.png
├── calib_index_repeat<i>.npy   # Calibration indices (NumPy)
├── set_recall_repeat<i>.pickle # Cached recall method results
├── set_prec_repeat<i>.pickle   # Cached precision method results
├── set_min_repeat<i>.pickle    # Cached min method results
├── ADiT_DSI_repeat<i>.pickle   # Cached ADiT-DSI results
└── ArbiTree_CQC_repeat<i>.pickle # Cached ArbiTree-CQC results
```

---

## CSV Tables

### `coverage_table.csv`

**Description**: Coverage rates (proportion of test samples where prediction set achieves the required power) for each method and confidence level.

**Format**:
```csv
Method,α=0.05,α=0.07,α=0.10,α=0.15,α=0.20
set_recall,0.967 ± 0.012,0.958 ± 0.015,...
set_prec,0.971 ± 0.010,0.962 ± 0.013,...
set_min,0.965 ± 0.014,0.955 ± 0.016,...
```

**Columns**:
- `Method`: Name of the conformal prediction method
- `α=<level>`: Mean coverage ± standard deviation across MC runs

**Interpretation**:
- For α=0.05, target coverage is 95%
- Values should be ≥ (1-α) for valid coverage
- Lower variance indicates more stable performance

---

### `set_size_table.csv`

**Description**: Average prediction set sizes for each method and confidence level.

**Format**:
```csv
Method,α=0.05,α=0.07,α=0.10,α=0.15,α=0.20
set_recall,45.234 ± 3.21,42.156 ± 2.89,...
set_prec,38.567 ± 2.45,35.890 ± 2.12,...
set_min,52.123 ± 4.56,48.789 ± 3.78,...
```

**Columns**:
- `Method`: Name of the conformal prediction method
- `α=<level>`: Mean set size ± standard deviation across MC runs

**Interpretation**:
- Smaller sets are better (more precise identification)
- Set size typically decreases as α increases (lower confidence)
- Compare with number of true sources and network size

---

### `time_cost_table.csv`

**Description**: Runtime in seconds for computing conformity scores and prediction sets on the test set.

**Format**:
```csv
Method,α=0.05,α=0.07,α=0.10,α=0.15,α=0.20
set_recall,12.34 ± 1.23,12.34 ± 1.23,...
set_prec,11.89 ± 1.15,11.89 ± 1.15,...
set_min,10.56 ± 0.98,10.56 ± 0.98,...
```

**Note**: Time cost is typically the same across α levels for the same method (computation is done once per run).

## PDF Visualizations

### `coverage_boxplot.pdf`

**Description**: Box plots comparing coverage distributions across methods and confidence levels.

**Contents**:
- X-axis: Confidence levels (α)
- Y-axis: Coverage rate
- Colors: Different methods
- Boxes: Distribution across MC runs
- Horizontal lines at (1-α) indicate target coverage

### `set_size_boxplot.pdf`

**Description**: Box plots comparing prediction set size distributions.

**Contents**:
- X-axis: Confidence levels (α)
- Y-axis: Average set size
- Colors: Different methods
- Boxes: Distribution across MC runs

## Cached Results (Pickle Files)

### `set_recall_repeat<i>.pickle`

**Description**: Cached results for the recall-based conformal method.

**Schema**:
```python
{
    'cover': np.ndarray,     # Shape: (n_alpha,) - coverage for each α
    'set_size': np.ndarray,  # Shape: (n_alpha,) - avg set size for each α
    'time_cost': float       # Runtime in seconds
}
```

### `set_prec_repeat<i>.pickle`

**Description**: Cached results for the precision-based conformal method.

**Schema**: Same as `set_recall_repeat<i>.pickle`

### `set_min_repeat<i>.pickle`

**Description**: Cached results for the min-probability conformal method.

**Schema**: Same as `set_recall_repeat<i>.pickle`

### `ADiT_DSI_repeat<i>.pickle`

**Description**: Cached results for the ADiT-DSI baseline.

**Schema**: Same as `set_recall_repeat<i>.pickle`

### `ArbiTree_CQC_repeat<i>.pickle`

**Description**: Cached results for the ArbiTree-CQC method.

**Schema**: Same as `set_recall_repeat<i>.pickle`

## NumPy Arrays

### `calib_index_repeat<i>.npy`

**Description**: Indices of samples assigned to the calibration set for MC run `i`.

**Usage**:
```python
import numpy as np
calib_idx = np.load('results/.../calib_index_repeat0.npy')
# calib_idx contains sample indices (0 to n_samples-1)
```

## Visualization Outputs

When running `visualization.py`, additional files are created:

### `visualizations/sample<i>_<method>_alpha<a>_pow<p>_<range>_sources.png`

**Description**: Full visualization showing:
- Network graph layout
- Initial infected nodes (blue)
- Initial recovered nodes (purple)
- Correctly identified sources (green)
- Falsely identified sources (orange)
- Missed sources (red)

### `visualizations/sample<i>_<method>_alpha<a>_pow<p>_<range>_sources_only.png`

**Description**: Simplified visualization showing only source identification:
- Correctly identified (green)
- Falsely identified (orange)
- Missed (red)

### `visualizations/sample<i>_<method>_alpha<a>_pow<p>_<range>_plain_graph.png`

**Description**: Plain network graph without source highlighting.

## Intermediate File: `res.pickle`

Located at: `SD-STGCN/output/test_res/<graph>/<test_exp>/res.pickle`

**Description**: SD-STGCN model predictions on the test set.

**Schema**:
```python
{
    'predictions': list,      # List of batches
                              # Each batch: (batch_size x n_nodes x 2) array
                              # [:, :, 0] = P(not source)
                              # [:, :, 1] = P(source)
    
    'ground_truth': list,     # List of batches
                              # Each batch: (batch_size x n_nodes) array
                              # One-hot encoding: 1 if node is source, 0 otherwise
    
    'inputs': list,           # List of batches
                              # Each batch: (batch_size x n_nodes) array
                              # Values: 0=susceptible, 1=infected, 2=recovered
                              # Represents initial state at earliest observation time
    
    'logits': list            # List of batches
                              # Each batch: (batch_size x n_nodes x 2) array
                              # Raw logits before softmax
}
```

**Usage Example**:
```python
import pickle

with open('SD-STGCN/output/test_res/highSchool/exp1/res.pickle', 'rb') as f:
    data = pickle.load(f)

# Flatten batches
all_preds = []
for batch in data['predictions']:
    for sample in batch:
        all_preds.append(sample)

# Get source probability for first sample
prob_source = all_preds[0][:, 1]  # P(source) for each node
```

## Verifying Results

### Check Coverage Validity

Coverage should be approximately ≥ (1-α):
- For α=0.05, expect coverage ≥ 0.95
- Some deviation is expected due to finite sample sizes

### Compare Set Sizes

Good methods have:
- Valid coverage (≥ 1-α)
- Small set sizes (efficient identification)
- Low variance (stable performance)

### Reproduce Results

Results are cached in pickle files. To re-run:
1. Delete the corresponding `*_repeat<i>.pickle` files
2. Re-run `main.py`

Or change `--mc_runs` to generate additional iterations.
